# 如何将Mock数据导入云数据库

## 步骤1：上传云函数

在微信开发者工具中：

1. 找到左侧云开发面板
2. 右键点击 `cloudfunctions/importMockData` 文件夹
3. 选择 **"上传并部署：云端安装依赖"**
4. 等待上传完成

## 步骤2：调用云函数导入数据

### 方法一：通过云开发控制台测试

1. 打开微信开发者工具的 **云开发** 控制台
2. 进入 **云函数** 页面
3. 找到 `importMockData` 函数
4. 点击 **测试** 按钮
5. 在测试参数中输入：
   ```json
   {
     "collectionName": "all"
   }
   ```
6. 点击 **运行测试**

### 方法二：在小程序代码中调用

在任意页面的 `.js` 文件中添加以下代码：

```javascript
// 导入所有数据
wx.cloud.callFunction({
  name: 'importMockData',
  data: {
    collectionName: 'all'
  },
  success: res => {
    console.log('导入成功:', res.result);
    wx.showToast({
      title: '数据导入成功',
      icon: 'success'
    });
  },
  fail: err => {
    console.error('导入失败:', err);
    wx.showToast({
      title: '导入失败',
      icon: 'error'
    });
  }
});
```

### 方法三：只导入特定集合

如果只想导入某个特定的数据集合，可以指定 `collectionName`：

```javascript
// 只导入用户数据
wx.cloud.callFunction({
  name: 'importMockData',
  data: {
    collectionName: 'users'  // 可选值: 'users', 'messages', 'chat_messages', 'profiles'
  },
  success: res => {
    console.log('导入成功:', res.result);
  }
});
```

## 步骤3：验证数据是否导入成功

1. 打开 **云开发控制台**
2. 进入 **数据库** 页面
3. 查看以下集合是否已创建并包含数据：
   - `users` - 用户列表（8条数据）
   - `messages` - 消息列表（4条数据）
   - `chat_messages` - 聊天记录（5条数据）
   - `profiles` - 个人资料（1条数据）

## 数据结构说明

### users 集合
存储用户信息，包含字段：
- `id`: 用户ID
- `name`: 姓名
- `age`: 年龄
- `city`: 城市
- `occupation`: 职业
- `interests`: 兴趣爱好（数组）
- `avatar`: 头像URL
- `bio`: 个人简介
- `isLiked`: 是否已喜欢
- `likesMe`: 是否喜欢我
- `createTime`: 创建时间

### messages 集合
存储消息列表，包含字段：
- `userId`: 对方用户ID
- `userName`: 对方姓名
- `userAvatar`: 对方头像
- `lastMessage`: 最后一条消息
- `time`: 时间
- `unreadCount`: 未读消息数
- `createTime`: 创建时间

### chat_messages 集合
存储聊天记录，包含字段：
- `senderId`: 发送者ID
- `receiverId`: 接收者ID
- `content`: 消息内容
- `time`: 时间
- `isSelf`: 是否是自己发送的
- `createTime`: 创建时间

### profiles 集合
存储个人资料，包含字段：
- `userId`: 用户ID
- `name`: 姓名
- `age`: 年龄
- `city`: 城市
- `occupation`: 职业
- `interests`: 所有兴趣标签
- `activeInterests`: 选中的兴趣标签
- `avatar`: 头像URL
- `photos`: 照片列表
- `bio`: 个人简介
- `visitors`: 访客数
- `likes`: 被喜欢数
- `matches`: 匹配数
- `createTime`: 创建时间
- `updateTime`: 更新时间

## 注意事项

1. **首次导入前**：确保已在云开发控制台中开通了云数据库
2. **权限设置**：默认情况下，新创建的集合权限为"仅创建者可读写"，如需修改请在云开发控制台中调整
3. **重复导入**：每次导入会先清空该集合的现有数据，再插入新数据
4. **数据量限制**：当前mock数据量较小，实际生产中可能需要分批导入大量数据

## 常见问题

**Q: 上传云函数时提示缺少依赖？**
A: 确保选择了"上传并部署：云端安装依赖"，而不是"上传并部署：所有文件"

**Q: 导入后看不到数据？**
A: 检查云开发控制台的环境是否正确，确保与小程序中配置的环境ID一致

**Q: 如何修改导入的数据？**
A: 可以直接修改 `cloudfunctions/importMockData/index.js` 中的mock数据数组，然后重新上传云函数并调用
